一、检索视频文件和字幕的逻辑
    1、当前一层文件夹，查找车牌收集视频，找不到车牌的用ScrapeError标记
    2、检查独立文件夹，当前文件夹是否只存在一种车牌，注意找不到车牌的（null也是一种车牌）
    3、检查每个jav的自身属性，中字、流出、破解等，组合成它的版本
    4、按车牌、版本划分CD，例如一个文件夹下有“流浪地球-C.mp4”“流浪地球-cd1.mp4”“流浪地球-cd2.mp4”，
        那么这个文件夹下的《流浪地球》有两个版本，第一个是有中字，第二个是无中字，第二个版本还有两个CD。
    5、收集外挂字幕，如果有与字幕同名的视频，则指定该视频的外挂字幕，如果不同名，且同车牌的视频较多，
        则给字幕用ScrapeError标记为【字幕无法确定归属】

二、在javdb上遍历车头页面
    1、访问首页，不存在404=>结束
    2、首页的最小车尾 比 车牌车尾 大多少，用40每页去预估在多少页
    3、检查预估的页面，如果比预估页面的最小车尾小，就往后推；如果比预估页面的最大车尾大，就往前推。否则就在预估页面找。
    4、往前或往后推，直至找到

三、用户命名规则
    1、不允许把是否中字这些
    

目标 不存在 => 重命名 true

目标 存在
    Windows （大小写不敏感）
        大小写不敏感，不同  => 不是自己，是另一个重复文件 => 不动 false
        大小写不敏感，相同  => 是自己，可能大小写不同
            大小写敏感，相同 => 是自己，完全相同 => 不动 true
            大小写敏感，不同 => 是自己，但大小写不同 => 重命名 true => 用一个临时文件move两次

    linux/mac （大小写敏感）
        大小写敏感，相同  => 是自己 => 不动 true
        大小写敏感，不同  => 不是自己，是另一个重复文件 => 不动 false
    
步骤1 重命名视频+字幕
    重命名视频失败 =>  视频文件名不变 => 字幕继续重命名，但重命名跟随老视频文件名
        => 重命名字幕失败 =>
            => 不进行步骤2 || 不进行步骤3+7
        => 重命名字幕成功 =>
            => 不进行步骤2 || 不进行步骤3+7

    重命名视频成功 => 视频文件名改变 => 字幕继续重命名，跟随新视频文件名
        => 重命名字幕失败
            => 不进行步骤2 || 不进行步骤3+7
        => 重命名字幕成功 =>
            => 可进行步骤2 || 可进行步骤3+7

步骤2 归类视频+字幕 与 步骤3+7互斥
    归类视频失败 => Dir不变 => 字幕不归类

    归类视频成功 => Dir改变 => 字幕归类
        字幕失败 => 仅报错字幕

步骤3 重命名或创建独立文件夹，如果执行步骤2，则跳过步骤3
    重命名已独立文件夹失败（步骤3.1） => Dir不变
        => 归类文件夹（步骤7）不进行
    
    创建新独立文件夹失败（步骤3.2） => Dir不变
        => 创建新独立文件夹失败
            => 归类文件夹（步骤7）不进行
        => 创建新独立文件夹成功
            => 移动视频失败
                => 不移动字幕 => 归类文件夹（步骤7）不进行
            => 移动视频成功，但移动字幕失败
                => 归类文件夹（步骤7）可进行
    

步骤7 ，如果执行步骤2，则跳过步骤7


四、同一电影的不同版本复杂情况
    假设现在文件夹下有多个流浪地球的视频文件，
        |根目录
            |
            |流浪地球(2019新春贺岁)
            |   |
            |   |流浪地球-cd1.mp4
            |   |
            |   |流浪地球-cd2.mp4
            |   |
            |   |花絮.zip（这个是用户自己收集的，要带着它一起整理）
            |
            |流浪地球(电影基地首发)
                |
                |流浪地球-中英双语字幕_1.mp4
                |
                |流浪地球-中英双语字幕_2.mp4


    目标是整理成以下效果，
        |根目录
            |
            |归类完成
                |
                |流浪地球 (2019)
                    |
                    |流浪地球-cd1.mp4
                    |
                    |流浪地球-cd2.mp4
                    |
                    |流浪地球 - ㊥-cd1.mp4
                    |
                    |流浪地球 - ㊥-cd2.mp4
                    |
                    |花絮写真.zip
    
    难点在于如何带着视频的老文件夹里的内容。
    这两个版本的流浪地球的目标文件夹是同一个，如果新建这个目标文件夹，如何把视频移动进去，就会丢弃老文件夹的内容。
    之前Python版的做法是检查老文件夹是不是一部电影的独立文件夹，如果是，直接移动这个文件夹，而不是新建。
    但是这么做，多个老文件夹会冲突。

    目前的想法是，初步检索文件夹中的视频后，把上述情况的电影标记一下，要求用户必须把同一电影的不同版本提前归集到一个文件夹下。

五、用户需要规范目录的情况
    （1）用户保持原文件夹
        选【无操作】，禁用文件结构
    （2）用户重命名文件夹，但还是在原来的祖父目录
        选【根目录】，文件结构写新文件夹
    （3）用户归类文件夹，在【归类完成】中
        选【根目录带归类完成】，文件结构写新文件夹
            如果视频在独立文件夹中，
                当前视频是家族中的最后一集，则移动文件夹
            当前视频不在独立文件夹中，
                不管视频是家族中的第几集，移动视频本身
    （4）用户归类文件，在【归类完成】中
        选【根目录带归类完成】，文件结构为空
    （5）用户自定义归类根目录
        选【自定义根目录】，文件结构为空
